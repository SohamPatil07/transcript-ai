# Codebase Index

## What this repo is

`Transcript AI` is a multi-surface YouTube transcript app with:

- a React web app
- an Electron desktop wrapper
- a Chrome side panel extension
- Netlify Functions for transcript extraction, summaries, and transcript chat

- `src/` - main React application used by the web app and desktop app

## Runtime ownership

### `src/`

- `src/main.jsx`
  - Main client entry point.
  - Renders the whole app.
  - Handles Clerk auth for web/desktop.
  - Switches to anonymous/embedded behavior for the extension.
  - Owns transcript history in `localStorage`.
  - Calls Netlify functions for `transcribe`, `summarize`, and `chat`.
  - Includes transcript list, profile UI, summary UI, and transcript chat UI.
- `src/utils.js`
  - YouTube URL parsing and normalization.
  - Date formatting.
  - Transcript download helper.
- `src/styles.css`
  - Global app styling for the shared web/desktop UI.

### `netlify/functions/`

- `netlify/functions/transcribe.js`
  - POST endpoint for transcript extraction.
  - Verifies Clerk tokens for web/desktop requests.
  - Allows extension requests through `X-Extension-Request`.
  - Calls Apify to fetch YouTube transcript data.
  - Normalizes transcript payloads into the frontend format.
- `netlify/functions/summarize.js`
  - POST endpoint for summary generation.
  - Verifies auth.
  - Reuses cached summaries when provided.
  - Delegates LLM work to shared helpers in `_rag.js`.
- `netlify/functions/chat.js`
  - POST endpoint for transcript Q&A.
  - Verifies auth.
  - Finds local transcript matches.
  - Sends selected chunks plus recent history to the LLM.
- `netlify/functions/_rag.js`
  - Shared helpers for:
  - CORS responses
  - transcript chunking
  - lexical chunk matching
  - summary generation
  - chat answer generation
  - Groq API calls

### `extension/`

- `extension/sidepanel.jsx`
  - Chrome side panel React entry point.
  - Detects the active YouTube tab.
  - Builds an embedded app URL with `embed=sidepanel` and `youtubeUrl=...`.
  - Loads the main app inside an iframe.
- `extension/sidepanel.html`
  - HTML entry for the extension side panel.
- `extension/sidepanel.css`
  - Extension-specific styling.
- `extension/vite.config.mjs`
  - Build config for the extension bundle.
- `extension/public/background.js`
  - Background service worker.
  - Configures the side panel to open on action click.
- `extension/public/manifest.json`
  - Extension manifest with permissions, CSP, and side panel config.
  - This file is generated by `scripts/generate-extension-manifest.mjs`.
- `extension/public/rules.json`
  - Declarative Net Request rules resource referenced by the manifest.
- `extension/public/logo-mark.svg`
  - Extension branding asset.

### `electron/`

- `electron/main.js`
  - Electron main process.
  - Creates the desktop window.
  - Hosts built static assets over local HTTP in production so Clerk OAuth can work.
  - Opens external non-app links in the system browser.
  - Sends renderer load errors to the frontend via IPC.
- `electron/preload.js`
  - Safe bridge exposing:
  - `isDesktop`
  - `apiBaseUrl`
  - load error subscription

### `scripts/`

- `scripts/desktop-dev.mjs`
  - Starts Vite, waits for the dev URL, then launches Electron against it.
- `scripts/generate-extension-manifest.mjs`
  - Reads env values and `package.json`.
  - Rebuilds `extension/public/manifest.json` with correct hosts and CSP.
- `scripts/production-build.bat`
  - Windows-oriented production build helper.
- `scripts/production-build.sh`
  - Unix shell production build helper.

## Request flow

### Transcript extraction

1. User submits a YouTube URL in `src/main.jsx`
2. Frontend calls `/.netlify/functions/transcribe`
3. `transcribe.js` verifies auth or extension header
4. `transcribe.js` calls Apify and normalizes the transcript
5. Frontend stores the transcript in local history

### Summary generation

1. Frontend sends transcript text to `/.netlify/functions/summarize`
2. `summarize.js` calls `generateSummary()` from `_rag.js`
3. `_rag.js` chunks the transcript and uses Groq for section summaries plus final synthesis

### Transcript chat

1. Frontend sends the question, transcript text, and recent messages to `/.netlify/functions/chat`
2. `chat.js` selects relevant chunks via `findTranscriptMatchesLocally()`
3. `_rag.js` sends those chunks to Groq and returns an answer plus source chunks

## Build entry points

- `npm run dev` - Netlify dev server for frontend plus functions
- `npm run dev:vite` - Vite frontend only
- `npm run build` - web app production build
- `npm run desktop:dev` - Vite plus Electron dev
- `npm run desktop:dist` - web build plus Electron installer build
- `npm run extension:prepare` - regenerate extension manifest
- `npm run extension:build` - build the extension bundle

## Key environment variables

- `VITE_CLERK_PUBLISHABLE_KEY` - Clerk frontend auth key
- `CLERK_SECRET_KEY` - Clerk backend auth key
- `APIFY_TOKEN` - transcript extraction API auth
- `APIFY_ACTOR_ID` - transcript extraction actor id
- `GROQ_API_KEY` - LLM API auth
- `GROQ_SUMMARY_MODEL` - optional summary model override
- `GROQ_CHAT_MODEL` - optional chat model override
- `VITE_API_BASE_URL` - frontend API base URL override
- `VITE_EXTENSION_API_BASE_URL` - extension backend base URL
- `VITE_EXTENSION_APP_URL` - extension iframe app URL
- `DESKTOP_API_BASE_URL` - desktop app backend base URL

## Current architectural shape

- The web app and desktop app share the same React frontend in `src/`.
- The extension is a thin wrapper around the same app, loaded in an iframe.
- Persistence is currently client-side via `localStorage`, partitioned by user id.
- Netlify Functions are the only backend code in active use.
- `supabase/` exists, but there are no active app-side integrations in the current source tree.
- `Transcript-AI-Logo.png` is the main branding asset used by the project.

## Quick "where to look" guide

- Change transcript UX: `src/main.jsx`, `src/styles.css`
- Change YouTube URL parsing: `src/utils.js`
- Change transcript extraction/backend normalization: `netlify/functions/transcribe.js`
- Change summary behavior or model prompts: `netlify/functions/_rag.js`
- Change transcript chat retrieval or answer prompts: `netlify/functions/chat.js`, `netlify/functions/_rag.js`
- Change extension behavior: `extension/sidepanel.jsx`, `extension/public/background.js`
- Change desktop shell behavior: `electron/main.js`, `electron/preload.js`
