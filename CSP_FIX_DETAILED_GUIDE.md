# Content Security Policy (CSP) Fix - Detailed Technical Guide

## The Problem: "This content is blocked" Error

When you open the Transcript AI extension on YouTube and see:
```
This content is blocked. Contact the site owner to fix the issue.
```

This is a **Content Security Policy (CSP) violation**. The browser is blocking the iframe from loading the Netlify app URL.

---

## Root Cause Analysis

### The CSP Violation
Your extension manifest has this Content Security Policy:

```json
"content_security_policy": {
  "extension_pages": "script-src 'self'; object-src 'self'; connect-src 'self' http://127.0.0.1:8888 https://api.clerk.com ...; frame-src 'self' http://127.0.0.1:8888 https://*.clerk.accounts.dev ..."
}
```

**The problem**: The `frame-src` directive doesn't include `https://ai-transcript.netlify.app`

When the iframe tries to load from Netlify, the CSP blocks it because:
1. Development URL: `http://127.0.0.1:8888` ✓ allowed
2. Production URL: `https://ai-transcript.netlify.app` ❌ **NOT allowed**

### Why This Happens
The extension's manifest is dynamically generated by `scripts/generate-extension-manifest.mjs`:

```javascript
const appBaseUrl = 
  env.VITE_EXTENSION_APP_URL ||
  env.VITE_EXTENSION_API_BASE_URL ||
  env.VITE_API_BASE_URL ||
  "http://127.0.0.1:8888";

const frameSrc = unique([
  "'self'",
  normalizeOrigin(appBaseUrl),  // ← This determines what URLs are allowed
  "https://*.clerk.accounts.dev",
  // ...
]);
```

**When building:**
- If `VITE_EXTENSION_APP_URL` is set → uses that URL in CSP ✓
- If not set → defaults to `http://127.0.0.1:8888` ❌

### Step 1: Verify .env.production

Open `.env.production`:

```env
# Production environment variables
VITE_API_BASE_URL=

# For extension: explicit production URLs
VITE_EXTENSION_API_BASE_URL=https://ai-transcript.netlify.app
VITE_EXTENSION_APP_URL=https://ai-transcript.netlify.app
```

**Must have:**
- ✓ `VITE_EXTENSION_APP_URL=https://ai-transcript.netlify.app`
- ✓ `VITE_EXTENSION_API_BASE_URL=https://ai-transcript.netlify.app`

**If missing**: Add them now and save.


This will:
1. Read `.env.production` (because `NODE_ENV=production`)
2. Get `VITE_EXTENSION_APP_URL=https://ai-transcript.netlify.app`

## CSP Security Explanation

### Why CSP Exists
Content Security Policy prevents:
- **Malware injection**: Blocks loading scripts from untrusted sources
- **Data theft**: Restricts where sensitive data can be sent
- **Clickjacking**: Prevents framing from unauthorized sites

### What CSP Directives Do

| Directive | Purpose |
|-----------|---------|
| `script-src 'self'` | Only load scripts from extension itself (no eval) |
| `connect-src ...` | Only connect to these APIs |
| `frame-src ...` | Only embed iframes from these domains |
| `object-src 'self'` | Only load objects from extension |

### Our CSP Configuration
```
script-src 'self'
  → Only extension code can run (secure)

connect-src 'self' http://127.0.0.1:8888 https://api.clerk.com https://*.clerk.accounts.dev
  → Connect to Netlify, Clerk for auth

frame-src 'self' https://ai-transcript.netlify.app https://accounts.google.com
  → Embed Netlify app and Google auth pages

object-src 'self'
  → No external objects (very restrictive = secure)
```

## Troubleshooting CSP Issues

### Issue: Still seeing "This content is blocked"

**Step 1**: Clear Chrome extension cache
```
chrome://extensions/ → Remove extension → Clear browsing data
```

**Step 2**: Verify manifest.json was rebuilt
```bash
# Check modification time
dir /A dist-extension\manifest.json

# Should be recent (after you ran build)
# If old, build wasn't executed
```

**Step 3**: Check NODE_ENV was set
```bash
# Run with explicit check:
node -e "console.log(process.env.NODE_ENV)"

# Should output: production
# If shows nothing, env not set correctly
```

**Step 4**: Check .env.production has URLs
```bash
# Verify file exists and has URLs
type .env.production | findstr VITE_EXTENSION_APP_URL
```

### Issue: Manifest generation failed

**Check** if script has errors:
```bash
node scripts/generate-extension-manifest.mjs
```

If error about missing .env.production:
```bash
# Create it if missing
type nul > .env.production
echo VITE_EXTENSION_APP_URL=https://ai-transcript.netlify.app >> .env.production
```

## CSP Headers in HTTP Responses

### Optional: Add CSP Headers to Netlify Functions

For extra security, you can also set CSP headers on API responses:

```javascript
// In netlify/functions/_rag.js
function createCorsHeaders(origin = "*") {
  return {
    "Content-Type": "application/json",
    "Access-Control-Allow-Origin": origin,
    "Access-Control-Allow-Headers": "Content-Type, Authorization",
    "Access-Control-Allow-Methods": "GET, POST, OPTIONS",
    // Optional: Add these for extra security
    "Content-Security-Policy": "default-src 'self'",
    "X-Content-Type-Options": "nosniff",
    "X-Frame-Options": "DENY",
  };
}
```

This prevents the API responses themselves from being framed or exploited.
